# Domain Events & Storming (05.01_ddd_domain_events.md)

**Purpose:** Model the initial domain structure of the Skill, identifying the core entities and the flow of events that occur during the architectural interview.

---

## 1. System Actors
> Identify the users, external systems, or agents that interact in the domain covered by this problem.
- **Developer / End User:** Initiates the Skill's execution, provides the project context by answering the interview, and validates the results.
- **LLM Engine (Artificial Intelligence):** Acts as the "interviewer agent", orchestrating the Skill's rules, interpreting context, and updating the state in the Markdown files.

## 2. Aggregates and Entities (Domain Entities)
> List the main aggregates and entities of this context.
- **Rules Engine (Domain Policies):** Static set formed by the `Skill.md` and the adjacent phase scripts. They act in unison as the "brain" of the operation, dictating logical validations and the interview flow.
- **Session State (`tracking_changelog.md`):** Dynamic aggregate that acts as the "living memory", tracking progress, completed methodologies, and the active topic.
- **Document Artifact (`.md` files):** The final delivery entity that undergoes iterative mutations being filled from the base templates.

## 3. Domain Commands and Events
> Map the triggering actions (Commands) and the results that affect the system (Events).

### 3.1. Standard Creation Flow (`new`)
- **Command:** `NewProject` (`/interview new <alias>`).
- **Domain Validations:** 
  1. Checks for the existence of `<alias>`.
  2. Conflict prevention: if the folder already exists, requires the user to choose between overwriting the current one, renaming the new one, or renaming the old one.
- **Generated Event:** `InterviewStarted` (Folder resolved, changelog initialized, first phase activated).

### 3.2. Contextual Creation Flow (`docs`)
- **Command:** `StartWithDocs` (`/interview docs <docs_path> <alias>`).
- **Domain Validations:** Validates the readability of the base at `<docs_path>` and applies the same conflict prevention rules as the `new` command.
- **Generated Event:** `ContextLoaded` (Knowledge base absorbed) and `InterviewStarted` (Session proactively started).

### 3.3. Continuous Interaction Flow
- **Command:** `AnswerQuestion` (Organic user response during the interview).
- **Domain Validations:** The Rules Engine evaluates whether the answer meets the requirement of the current phase.
- **Generated Event:** `ArtifactUpdated` (document receives content) and `StateTracked` (pointer advances).

### 3.4. Resumption Flow (`resume`)
- **Command:** `ResumeProject` (`/interview resume <path>`).
- **Domain Validations:** Validates the structural integrity of the `tracking_changelog.md` and reads all documents from `<path>`.
- **Generated Event:** `InterviewResumed` (AI recovers context and launches the next pending question).

### 3.5. Audit Flow (`validate`)
- **Command:** `ValidateProject` (`/interview validate <path>`).
- **Domain Validations:** Scans all files in `<path>` looking for gaps or inconsistencies in the generated architecture.
- **Generated Event:** `ValidationReportGenerated` (`00_validation_report.md` file physically created).

## 4. Asynchronous Events / Reactions
> If applicable, list which events trigger asynchronous reactions in other contexts or systems.
- **N/A:** The Skill operates strictly in the local environment (Markdown file manipulation) and has no background jobs, webhooks, or triggers to external APIs after domain events are consolidated.
