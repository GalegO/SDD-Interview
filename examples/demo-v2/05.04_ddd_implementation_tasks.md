## 2. Backend Tasks (`[CORE]`)
> Enumerated list of atomic steps. If the issue is only frontend/db, indicate a Pass-Through.
1. **Query Routine:** Read the map `reference/methodology-map.md` in conjunction with the active script from the `phases/` folder.
2. **Artifact Initialization:** Read the mold in `Templates/` and materialize the base physical structure in the destination directory before the first question.
3. **Logical Validation:** Challenge shallow user responses (without inferring facts in `new` mode; and presenting text-based drafts in `docs` mode).
4. **Artifact Writing (I/O):** Write the validated response directly into the respective generated Markdown document.
5. **Transactional Checkpoint:** Update `current_topic_id` and `covered_topics` in the changelog immediately after covering a topic.
6. **Artifact Closure:** When covering all topics of a file, finalize it and add it to the global `generated_files` log.
7. **Methodology Completion:** When all files of a methodology are generated, add its respective ID to the `completed_methodologies` array to unlock progression.
