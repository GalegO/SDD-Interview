# Initial Discovery

## Template

Read [Templates/00.01_initial_discovery_project_briefing.md](Templates/00.01_initial_discovery_project_briefing.md) before generating [00.01_initial_discovery_project_briefing.md](00.01_initial_discovery_project_briefing.md).

## Required topics

Start with `discovery.source_material`. Ask one question before any other: "Do you have any documentation, request, ticket or issue number, link, file, or other reference that can help direct this interview?" If the user does, ask them to provide the available material or its accessible path. Read it according to `reference/docs-mode.md`, use it only as interview context, and do not write ticket or issue metadata into the generated document. If the user does not have material, mark the topic covered and continue without blocking.

Then cover `discovery.project_name` (name or codename), `discovery.macro_objective` (purpose), `discovery.platform` (web, desktop, mobile, or mixed), `discovery.tech_stack` (intended technologies without inference), `discovery.complexity` (complexity and main challenge), and `discovery.branding` (tone and visual perception).
