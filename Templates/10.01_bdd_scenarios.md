## Scenario 1: Happy Path
> What is the ideal and error-free flow?
- **Given:** That the system/user is in the initial state X.
- **When:** Action Y is triggered with the correct data.
- **Then:** Result Z must occur (e.g., database updates / system state changes and success confirmation / green message displays).

## Scenario 2: Logical Exceptions Handling
> What happens if there is a payload validation error?
- **Given:** That user X has no balance.
- **When:** Trying to complete transaction Y.
- **Then:** The operation must be aborted with a validation error / HTTP status 400 and the system should present an error state (UI message / CLI error code).

## Scenario 3: Concurrency Restrictions
> Critical cases that must be validated by TDD.
- **Given:** That 2 simultaneous requests occur on the same row / resource.
- **When:** Both try to execute an UPDATE.
- **Then:** The database / system must apply a lock / optimistic control and only one should succeed, triggering a 409 Conflict / concurrency exception on the other.
